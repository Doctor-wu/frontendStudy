## git基础工作流程

每一个GIT仓库都有3个区

- 工作区：写代码
- 暂存区：临时存放每一次修改的代码，但是并没有生成历史记录
- 历史区：存放所有的历史版本



### 工作流程

- 创建GIT仓库

  git init

- 编写代码并将工作代码提交到暂存区

  git add -A/. (可以指定提交某些文件)

- 把暂存区还没有提交的内容提交到历史区，生成一个历史版本

  git commit -m['备注']

  每一次区域间的信息提交不会删除原有区域的内容，这样保证下一次提交，只是把区域间对比不一样的内容提交，而不是所有



### 细节问题

git config 查看git 配置

.gitignore 可以配置什么文件不提交

git log 查看历史版本
